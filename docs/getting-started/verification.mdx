# Installation Verification

After completing installation and configuration, verify that all components work correctly.

## Verification Checklist

Use this complete checklist to ensure everything is configured correctly.

### ✅ 1. Verify Lando Services

Check that all containers are running:

```bash
lando ps
```

**Expected Output:**

```
SERVICE      STATUS    URLS                              PORTS
appserver    RUNNING   https://zuora-workflows.lndo.site 443->80, 8080->80
database     RUNNING                                      3306
cache        RUNNING                                      6379
```

<Warning>
If any service shows "EXITED", restart with `lando rebuild -y`
</Warning>

### ✅ 2. Verify Laravel Application

#### PHP and Laravel Version

```bash
lando php -v
lando artisan --version
```

**Expected Output:**
- PHP 8.4+
- Laravel 12.x

#### Application Health

```bash
lando artisan about --only=environment
```

**Verify Output:**

```
Environment
---------------------------------------------------------------------------
Application Name: Zuora Workflows
Laravel Version: 12.x.x
PHP Version: 8.4.0
Composer Version: 2.x.x
Environment: local
Debug Mode: ENABLED
URL: https://zuora-workflows.lndo.site
Maintenance Mode: OFF
```

#### Application Key

```bash
lando artisan key:generate
```

If key doesn't exist, it will be generated automatically.

### ✅ 3. Verify Database

#### Database Connection

```bash
lando mariadb
```

If you can access MariaDB shell, connection works.

#### Tables Created

```bash
lando mariadb -e "SHOW TABLES;"
```

**Required Tables:**

```
+------------------------+
| Tables_in_zuora_workflows |
+------------------------+
| customers              |
| users                  |
| workflows              |
| tasks                  |
| settings               |
| jobs                   |
| failed_jobs            |
| job_batches            |
| migrations             |
+------------------------+
```

#### Migrations Status

```bash
lando artisan migrate:status
```

All migrations should show "Yes" in Batch column.

### ✅ 4. Verify Storage Permissions

```bash
# Check storage permissions
ls -la storage/ -la

# Check cache permissions
ls -la bootstrap/cache/ -la
```

**Required Permissions:**

- Directory: `775` or `770`
- File: `664` or `660`

**Fix if needed:**

```bash
chmod -R 775 storage bootstrap/cache
chmod -R 775 public
```

### ✅ 5. Verify Frontend Assets

#### Compiled Assets

```bash
ls -la public/build/
```

You should see CSS and JS files compiled by Vite:

```
-rw-r--r-- 1 user user 12345 Dec 27 14:30 assets/
-rw-r--r-- 1 user user 67890 Dec 27 14:30 app.css
-rw-r--r-- 1 user user 123456 Dec 27 14:30 app.js
```

#### Test Vite Dev Server

```bash
yarn run dev
```

Vite should start and show message:

```
  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

### ✅ 6. Verify Queue System

#### Queue Connection

```bash
# Check queue configuration
lando artisan tinker --execute="echo config('queue.default');"
```

Should show: `database` (or `sync`, `redis`)

#### Test Queue Worker

```bash
# Start queue worker (single job test)
lando artisan queue:work --once
```

Should complete without errors.

#### Queue Monitoring

```bash
# Monitor queued jobs
lando artisan queue:monitor database
```

Expected output:

```
Queue     Size    Status
database  0       OK
```

#### Failed Jobs

```bash
# Check failed jobs
lando artisan queue:failed
```

Should show: `No failed jobs` (or low number in development)

### ✅ 7. Verify Scheduler

#### List Scheduled Tasks

```bash
lando artisan schedule:list
```

**Expected Output:**

```
Next Due: 2025-12-27 14:00:00 UTC
┌────────────────────────────────────────────────────────────────────────────────────────┐
│ Command                                                                   │
├────────────────────────────────────────────────────────────────────────────────────────┤
│ php artisan app:sync-workflows --all │
└────────────────────────────────────────────────────────────────────────────────────────┘
```

#### Test Scheduler

```bash
# Run scheduler manually
lando artisan schedule:run
```

Should show task execution message.

### ✅ 8. Verify Filament Admin Panel

#### Dashboard Access

Navigate to: `https://zuora-workflows.lndo.site/admin`

**Verify:**
- ✅ Login page loaded correctly
- ✅ Filament design displayed
- ✅ Login form working
- ✅ Redirect to dashboard after login

#### Login Test

1. Create user account
2. Login
3. Verify dashboard access

**Post-Login Verification:**
- ✅ Main dashboard loaded
- ✅ Sidebar menu visible
- ✅ Resources accessible (Customers, Workflows, Tasks)
- ✅ User profile menu working

#### Debugbar (Development)

If `DEBUGBAR_ENABLED=true`, verify:

- ✅ Debug bar at bottom of page
- ✅ Executed queries displayed
- ✅ Execution time shown
- ✅ Memory usage shown

### ✅ 9. Verify API Endpoints

#### Test Authentication

```bash
# Create user and get token (from Filament)
curl -X GET https://zuora-workflows.lndo.site/api/user \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/json"
```

Should return user JSON data.

#### Test Zuora Token Endpoint

```bash
curl -X GET "https://zuora-workflows.lndo.site/api/zuora/token" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/json"
```

Should return `{ "accessToken": "..." }` or error 500 if credentials missing.

### ✅ 10. Verify Logs

#### Laravel Log

```bash
# Last 50 log lines
tail -n 50 storage/logs/laravel.log

# Follow logs in real time
tail -f storage/logs/laravel.log
```

**Verify:**
- ✅ No critical errors
- ✅ Loading information
- ✅ Logged queries (if debug active)

#### PHP Error Log

```bash
# Check PHP errors
tail -n 50 storage/logs/laravel.log | grep -i error
```

Should show few or no errors.

## Key Functionality Tests

### Test 1: Customer Creation

1. Access admin panel
2. Go to Customers → New Customer
3. Create customer with test data
4. Verify save

```bash
# Verify customer in database
lando mariadb -e "SELECT * FROM customers;"
```

### Test 2: Workflow Synchronization

1. Open created customer
2. Click "Sync Workflows"
3. Verify job dispatched

```bash
# Check queued job
lando mariadb -e "SELECT * FROM jobs;"
```

4. Start queue worker if needed:
   ```bash
   lando artisan queue:work
   ```

5. Verify synced workflows:
   ```bash
   lando mariadb -e "SELECT * FROM workflows;"
   ```

### Test 3: Workflow Visualization

1. Go to Workflows → List
2. Click on a workflow
3. Verify details visualization
4. Verify task relation manager
5. Verify JSON export

### Test 4: Task Extraction

1. Open a workflow
2. Go to Tasks tab
3. Verify tasks extracted from JSON
4. Verify task filters work

## Performance Baseline

### Loading Benchmark

Use browser DevTools to measure:

```javascript
// In browser console
performance.mark('start')
location.reload()
performance.mark('end')
performance.measure('load', 'start', 'end')
performance.getEntriesByName('load')[0].duration
```

**Target:**
- Dashboard < 2s
- Workflow List < 1.5s
- Workflow Detail < 1s

### Database Query Performance

```bash
# Enable query logging (development only)
lando artisan tinker --execute="DB::listen(function(\$query) { dump(\$query->sql); });"

// Execute some actions in the app
// Check logged queries
```

**Target:**
- < 50 queries per page
- No N+1 queries
- Indexes used where necessary

## Basic Troubleshooting

### If Something Doesn't Work

#### 1. Clear Cache

```bash
lando artisan optimize:clear
```

#### 2. Rebuild Container

```bash
lando rebuild -y
```

#### 3. Check Logs

```bash
lando logs -f
```

#### 4. Reset Application (ATTENTION)

```bash
# Complete reset - loses all data!
lando artisan migrate:fresh
lando artisan config:clear
lando artisan cache:clear
```

If all checks above pass ✅, your installation is complete and functional!
