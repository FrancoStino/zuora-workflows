# Installation

This guide walks you through installing Zuora Workflow Manager using Lando, which provides a containerized development environment with all dependencies pre-configured.

## Requirements

Before you begin, ensure you have the following installed:

| Requirement | Version | Link |
|-------------|---------|------|
| Lando | Latest | [lando.dev](https://lando.dev) |
| Docker | 20.0+ | [docker.com](https://www.docker.com/) |
| Node.js | 20.19+ | [nodejs.org](https://nodejs.org/) |
| Yarn | Latest | [yarnpkg.com](https://yarnpkg.com/) |

<Note>
Lando provides PHP 8.4, MariaDB 11.4, Nginx, Redis 7.0, and Xdebug pre-configured. It eliminates "works on my machine" problems by using Docker containers.
</Note>

## Installation steps

### Step 1: Clone the repository

```bash
git clone https://github.com/FrancoStino/zuora-workflows.git
cd zuora-workflows
```

### Step 2: Start Lando

```bash
lando start
```

This will automatically:
- Start PHP 8.4, MariaDB 11.4, Nginx, and Redis containers
- Run `lando composer install`
- Configure the development environment

### Step 3: Setup environment

```bash
# Copy environment file
cp .env.example .env

# Generate application key
lando artisan key:generate

# Run migrations
lando artisan migrate

# Install frontend dependencies (use yarn globally, not via lando)
yarn install

# Build frontend assets
yarn run build
```

### Step 4: Access the dashboard

Navigate to `https://zuora-workflows.lndo.site` and create your admin account.

<Warning>
Use Lando's Nginx URL (`https://zuora-workflows.lndo.site`), not port 8000.
</Warning>

### Step 5: Configure Zuora credentials

After login:

1. Go to **Customers** in the sidebar
2. Create a new customer
3. Enter Zuora API credentials:
   - **Client ID** - Your Zuora OAuth client ID
   - **Client Secret** - Your Zuora OAuth client secret
   - **Base URL** - `https://rest.zuora.com` (or `https://rest.test.zuora.com` for sandbox)
4. Save the customer
5. Click **Sync Workflows** to sync workflows from Zuora

## Quick commands

Common Lando commands for development:

```bash
lando artisan migrate                # Run migrations
lando artisan queue:work             # Start queue worker (database/redis only)
lando test                           # Run tests
lando logs -f                        # View logs
lando mariadb                        # Database access
```

## Troubleshooting

### Lando not starting

If Lando fails to start:

```bash
# Rebuild Lando containers
lando rebuild -y

# Check Docker is running
docker ps
```

### Permission errors

If you encounter permission errors:

```bash
# Fix storage permissions
chmod -R 775 storage bootstrap/cache
```

### Database connection errors

Verify your `.env` database settings match Lando's configuration:

```env
DB_CONNECTION=mysql
DB_HOST=database
DB_PORT=3306
DB_DATABASE=zuora_workflows
DB_USERNAME=zuora_workflows
DB_PASSWORD=zuora_workflows
```
