# Getting Started

This guide will help you set up and run Zuora Workflow Manager on your local development environment.

## Prerequisites

Before you begin, ensure you have the following installed:

### Required Software

| Software | Version | Installation Link |
|----------|---------|-------------------|
| [Lando](https://lando.dev) | Latest | [lando.dev](https://lando.dev/) |
| [Docker](https://www.docker.com/) | 20.0+ | [docker.com](https://www.docker.com/) |
| [Node.js](https://nodejs.org/) | 24.0+ | [nodejs.org](https://nodejs.org/) |
| [Yarn](https://yarnpkg.com/) | Latest | [yarnpkg.com](https://yarnpkg.com/) |
| [Git](https://git-scm.com/) | 2.0+ | [git-scm.com](https://git-scm.com/) |

<div className="warning">
  <p><strong>Important:</strong> Lando requires Docker to be installed and running before you can start the development environment.</p>
</div>

### Lando Stack

When you start Lando, it will automatically provision:
- **PHP 8.4** - Latest stable PHP version
- **MariaDB 11.4** - Database server
- **Nginx** - Web server
- **Redis 7.0** - Caching and queue storage
- **Xdebug** - Debugging tool

### Zuora API Credentials

You will need access to your Zuora account to obtain OAuth 2.0 credentials:

1. Navigate to Zuora Admin Console
2. Go to Settings → OAuth Clients
3. Create a new OAuth client with "Client Credentials" grant type
4. Note down:
   - **Client ID**
   - **Client Secret**
   - **Base URL** (Production: `https://rest.zuora.com`, Sandbox: `https://rest.test.zuora.com`)

## Installation

### Step 1: Clone Repository

```bash
git clone https://github.com/FrancoStino/zuora-workflows.git
cd zuora-workflows
```

### Step 2: Start Lando

```bash
lando start
```

This command will:
- Start all Docker containers (PHP, MariaDB, Nginx, Redis)
- Run `lando composer install` automatically
- Configure the development environment
- Generate application URL: `https://zuora-workflows.lndo.site`

<div className="info">
  <p><strong>Tip:</strong> Use <code>lando info</code> to view all available services and their URLs.</p>
</div>

### Step 3: Setup Environment

```bash
# Copy environment file
cp .env.example .env

# Generate application key
lando artisan key:generate

# Run database migrations
lando artisan migrate

# Install frontend dependencies
yarn install

# Build frontend assets
yarn run build
```

### Step 4: Access Application

Open your browser and navigate to:

```
https://zuora-workflows.lndo.site
```

You will be redirected to the setup wizard if this is your first visit.

## Initial Setup

### 1. Create Admin Account

On first launch, you'll see the setup wizard:

1. Enter your admin user details:
   - **Name**: Your full name
   - **Email**: Admin email address
   - **Password**: Secure password (min 8 characters)
2. Click "Create Account"
3. You'll be logged in automatically as super admin

### 2. Configure OAuth (Optional)

If you want to enable Google OAuth login:

1. Navigate to **Settings** → **General Settings**
2. Scroll to **OAuth Configuration** section
3. Enable OAuth by toggling the switch
4. Configure:
   - **Allowed Domains**: Comma-separated list of allowed email domains (e.g., `example.com,company.org`)
   - **Google Client ID**: OAuth client ID from Google Cloud Console
   - **Google Client Secret**: OAuth client secret from Google Cloud Console
5. Click **Save**

<div className="warning">
  <p><strong>Security Note:</strong> OAuth secrets are encrypted in the database using Laravel's encryption (APP_KEY).</p>
</div>

### 3. Add Customer and Configure Zuora

After setup, you need to configure a customer with Zuora credentials:

1. Go to **Customers** in the sidebar
2. Click **Create Customer**
3. Fill in customer details:
   - **Name**: Customer/company name
   - **Client ID**: Zuora OAuth client ID
   - **Client Secret**: Zuora OAuth client secret
   - **Base URL**: Select from dropdown:
     - `https://rest.zuora.com` (Production)
     - `https://rest.test.zuora.com` (Sandbox)
4. Click **Save**

### 4. Sync Workflows

After creating a customer:

1. Click on the customer record
2. Click **Sync Workflows** button in the header
3. This will queue a background job to fetch workflows from Zuora
4. Navigate to **Workflows** to see synchronized workflows

## Starting Queue Worker

For background job processing (workflow syncs, etc.), you need to start a queue worker:

```bash
# Start queue worker (recommended)
lando queue

# OR start scheduler for automatic hourly sync
lando schedule
```

<div className="info">
  <p><strong>Development Tip:</strong> For development, you can also use <code>QUEUE_CONNECTION=sync</code> in your .env file to process jobs immediately without a queue worker.</p>
</div>

## Troubleshooting

### Issue: "Site can't be reached" or "Connection refused"

**Solution**: Ensure Docker and Lando are running:
```bash
docker ps
lando start
```

### Issue: Migration errors

**Solution**: Clear database and re-run migrations:
```bash
lando artisan migrate:fresh
```

### Issue: Assets not loading (404 errors)

**Solution**: Rebuild frontend assets:
```bash
yarn run build
lando artisan config:clear
lando artisan cache:clear
```

### Issue: Queue jobs not processing

**Solution**: Ensure queue worker is running:
```bash
lando queue
```

Or use `QUEUE_CONNECTION=sync` in `.env` for development.

### Issue: SSL certificate errors

**Solution**: Trust Lando's SSL certificate:
```bash
lando security-certs
```

Then follow the instructions for your browser.

## Development Workflow

### Running Tests

```bash
# Run all tests
lando test

# Run specific test file
lando artisan test tests/Feature/SyncWorkflowsTest.php

# Run tests with coverage
lando artisan test --coverage

# Verbose output
lando artisan test -v
```

### Database Access

```bash
# Access MariaDB command line
lando mariadb

# View queue table
lando mariadb -e "SELECT * FROM jobs;"

# View failed jobs
lando mariadb -e "SELECT * FROM failed_jobs;"
```

### Viewing Logs

```bash
# View all Lando logs
lando logs -f

# View application logs
lando exec appserver tail -f storage/logs/laravel.log

# Filter for specific errors
lando exec appserver grep -i "error" storage/logs/laravel.log
```
