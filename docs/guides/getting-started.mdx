# Getting Started

This guide will help you set up and run Zuora Workflow Manager on your local development environment.

## Prerequisites

<Alert type="warning">
  <strong>Important:</strong> Lando requires Docker to be installed and running before you can start the development environment.
</Alert>

### Required Software

<RequirementsTable
  requirements={[
    { name: 'Lando', version: 'Latest', url: 'https://lando.dev', required: true },
    { name: 'Docker', version: '20.0+', url: 'https://www.docker.com/', required: true },
    { name: 'Node.js', version: '24.0+', url: 'https://nodejs.org/', required: true },
    { name: 'Yarn', version: 'Latest', url: 'https://yarnpkg.com/', required: true },
    { name: 'Git', version: '2.0+', url: 'https://git-scm.com/', required: true }
  ]}
/>

### Lando Stack

When you start Lando, it will automatically provision:
- **PHP 8.4** - Latest stable PHP version
- **MariaDB 11.4** - Database server
- **Nginx** - Web server
- **Redis 7.0** - Caching and queue storage
- **Xdebug** - Debugging tool

### Zuora API Credentials

You will need access to your Zuora account to obtain OAuth 2.0 credentials:

<Steps steps={[
  'Navigate to Zuora Admin Console',
  'Go to Settings â†’ OAuth Clients',
  'Create a new OAuth client with "Client Credentials" grant type',
  'Note down: Client ID, Client Secret, Base URL (Production: https://rest.zuora.com, Sandbox: https://rest.test.zuora.com)'
]} />

## Initial Setup

### 1. Create Admin Account

On first launch, you'll see the setup wizard:

<Steps steps={[
  'Enter your admin user details (Name, Email, Password)',
  'Click "Create Account"',
  'You\'ll be logged in automatically as super admin'
]} />

### 2. Configure OAuth (Optional)

If you want to enable Google OAuth login:

<Steps steps={[
  'Navigate to Settings â†’ General Settings',
  'Scroll to OAuth Configuration section',
  'Enable OAuth by toggling the switch',
  'Configure: Allowed Domains, Google Client ID, Google Client Secret',
  'Click Save'
]} />

<Alert type="warning">
  <strong>Security Note:</strong> OAuth secrets are encrypted in the database using Laravel's encryption (APP_KEY).
</Alert>

### 3. Add Customer and Configure Zuora

<Steps steps={[
  'Go to Customers in the sidebar',
  'Click Create Customer',
  'Fill in customer details (Name, Client ID, Client Secret, Base URL)',
  'Select Base URL from dropdown (Production or Sandbox)',
  'Click Save'
]} />

### 4. Sync Workflows

<Steps steps={[
  'Click on the customer record',
  'Click Sync Workflows button in the header',
  'This will queue a background job to fetch workflows from Zuora',
  'Navigate to Workflows to see synchronized workflows'
]} />

## Starting Queue Worker

For background job processing (workflow syncs, etc.), you need to start a queue worker:

<CodeBlock
  title="Queue Worker Commands"
  tabs={[
    {
      name: 'Queue Worker',
      code: 'lando queue',
      language: 'bash'
    },
    {
      name: 'Scheduler',
      code: 'lando schedule',
      language: 'bash'
    }
  ]}
/>

<Alert type="info">
  <strong>Development Tip:</strong> For development, you can also use <Code inline>QUEUE_CONNECTION=sync</Code> in your .env file to process jobs immediately without a queue worker.
</Alert>
