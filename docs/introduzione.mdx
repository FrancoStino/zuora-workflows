# Introduzione a Zuora Workflow Manager

Zuora Workflow Manager è un'applicazione web costruita con Laravel 12 e Filament 4.2 che semplifica la gestione dei workflow Zuora attraverso un'interfaccia amministrativa moderna e intuitiva.

## Perché Zuora Workflow Manager?

La gestione dei workflow Zuora può essere complessa, specialmente quando si lavora con:
- Multiple organizzazioni customer
- Sincronizzazione manuale dei dati
- Visualizzazione limitata dei workflow
- Mancanza di automazione

Zuora Workflow Manager risolve questi problemi fornendo:

### 1. Sincronizzazione automatica
- Sync schedulato ogni ora (configurabile)
- Sync manuale on-demand tramite UI
- Elaborazione in background con code
- Retry automatico in caso di errori

### 2. Multi-tenant
- Gestione di multiple organizzazioni customer
- Credenziali Zuora separate per ogni customer
- Isolamento completo dei dati
- Sicurezza con encryption delle credenziali

### 3. Visualizzazione avanzata
- Dashboard Filament ricca e responsive
- Visualizzazione grafica dei workflow
- Dettagli completi dei task
- Export JSON dei workflow

### 4. Gestione task
- Estrazione automatica dei task dal JSON
- Filtri avanzati per tipo, priorità e stato
- Visualizzazione dettagliata con slide-over
- Relazioni tra task e workflow

## Architettura

L'applicazione è costruita seguendo i principi SOLID e utilizza:

### Service Layer
- **ZuoraService**: Gestione autenticazione OAuth 2.0 e chiamate API
- **WorkflowSyncService**: Orchestrazione sincronizzazione workflow
- **OAuthService**: Integrazione Google OAuth per login

### Models
- **Customer**: Organizzazioni con credenziali Zuora
- **Workflow**: Workflow sincronizzati da Zuora
- **Task**: Task estratti dai workflow
- **User**: Utenti dell'applicazione con RBAC

### Background Jobs
- **SyncCustomersJob**: Job per sincronizzazione workflow customer
- Retry logic con backoff esponenziale
- Progress tracking con Moox Jobs

### Filament Resources
- **CustomerResource**: Gestione customer
- **WorkflowResource**: Visualizzazione workflow
- **TaskResource**: Gestione task
- **UserResource**: Gestione utenti
- **RoleResource**: Gestione ruoli e permessi

## Flusso di lavoro tipico

1. **Setup iniziale**
   - Installazione con Lando
   - Configurazione ambiente
   - Creazione admin account

2. **Configurazione customer**
   - Aggiunta customer nella UI
   - Inserimento credenziali Zuora (Client ID, Secret, Base URL)
   - Validazione credenziali

3. **Sincronizzazione workflow**
   - Click su "Sync Workflows" nella UI
   - Job accodato per elaborazione background
   - Download workflow da Zuora API
   - Salvataggio nel database locale
   - Estrazione automatica task

4. **Visualizzazione e gestione**
   - Navigazione workflow nella dashboard
   - Filtri e ricerca avanzata
   - Visualizzazione grafica
   - Export JSON
   - Dettagli task

5. **Monitoraggio**
   - Dashboard Moox Jobs per stato job
   - Log applicazione per troubleshooting
   - Health check sistema

## Tecnologie utilizzate

### Backend
- **Laravel 12**: Framework PHP moderno
- **PHP 8.4**: Ultima versione PHP
- **MariaDB 11.4**: Database relazionale
- **Redis 7.0**: Cache e code

### Frontend
- **Filament 4.2**: Admin panel framework
- **Tailwind CSS 4**: Utility-first CSS
- **Alpine.js**: JavaScript framework leggero
- **@joint/layout-directed-graph**: Visualizzazione grafi

### DevOps
- **Lando**: Ambiente di sviluppo containerizzato
- **Docker**: Containerizzazione
- **GitHub Actions**: CI/CD
- **Vite 7**: Build tool moderno

### Packages chiave
- **Filament Shield**: RBAC per Filament
- **Spatie Laravel Settings**: Gestione settings
- **Moox Jobs**: Monitoraggio job
- **Filament Socialite**: OAuth login
